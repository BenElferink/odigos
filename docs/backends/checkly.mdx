### Configuring Destination Fields

<Accordion title="Supported Signals:">
  ‚úÖ Traces
  ‚ùå Metrics
  ‚ùå Logs
</Accordion>

- **CHECKLY_ENDOINT** `string` : OTLP gRPC Endpoint. The format is `host:port`, host is required, port is optional and defaults to the default OTLP gRPC port `4317`. Do not include the path `/v1/traces`.
  - This field is required
- **CHECKLY_API_KEY** `string` : API Key.
  - This field is required
- **CHECKLY_WITH_FILTER** `boolean` : Filter Traces. If checked, only traces with the `checkly="true"` attribute will be exported to Checkly.
  - This field is optional

<Note>
  By default, Odigos will export all traces to Checkly.

  If you want to filter traces, you can toggle `CHECKLY_WITH_FILTER` to `true` and add the `checkly="true"` attribute to the traces you want to export to Checkly.
</Note>

### Adding Destination to Odigos

There are two primary methods for configuring destinations in Odigos:

##### **Using the UI**

<Steps>
  <Step>
    Use the [Odigos CLI](https://docs.odigos.io/cli/odigos_ui) to access the UI
    ```bash
    odigos ui
    ```
  </Step>
  <Step>
    Click on `Add Destination`, select `Checkly` and follow the on-screen instructions
  </Step>
</Steps>

##### **Using Kubernetes manifests**

<Steps>
  <Step>
    Save the YAML below to a file (e.g. `checkly.yaml`)
    ```yaml
    apiVersion: odigos.io/v1alpha1
    kind: Destination
    metadata:
      name: checkly-example
      namespace: odigos-system
    spec:
      data:
        CHECKLY_ENDOINT: <OTLP gRPC Endpoint>
        # Note: The commented fields below are optional.
        # CHECKLY_WITH_FILTER: <Filter Traces>
      destinationName: checkly
      secretRef:
        name: checkly-secret
      signals:
      - TRACES
      type: checkly

    ---

    apiVersion: v1
    data:
      CHECKLY_API_KEY: <Base64 API Key>
    kind: Secret
    metadata:
      name: checkly-secret
      namespace: odigos-system
    type: Opaque
    ```
  </Step>
  <Step>
    Apply the YAML using `kubectl`
    ```bash
    kubectl apply -f checkly.yaml
    ```
  </Step>
</Steps>
